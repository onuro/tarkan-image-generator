# Tarkan — AI Image Generator

A full-stack AI image generation app built with Next.js, Convex, and Google's Gemini/Imagen models. Generate images from text prompts with style presets, reference images, prompt enhancement, and more.

## Features

- **Text-to-Image Generation** with multiple Google AI models (Imagen 4, Gemini 3 Pro, Gemini 3.1 Flash, Gemini 2.5 Flash)
- **Prompt Enhancement** — auto-expands short prompts using Gemini 2.5 Flash
- **Multi-Reference Images** — attach up to 14 reference images per generation (upload, paste, or pick from saved library)
- **@Mention Autocomplete** — type `@` in the prompt to reference attached images by label (`@img1`, `@img2`, etc.) with thumbnail previews
- **11 Style Presets** — photorealistic, 3D render, anime, oil painting, watercolor, cinematic, and more
- **Aspect Ratio Control** — auto, 1:1, 16:9, 9:16, 4:3, 3:4
- **Batch Generation** — generate 1–4 images per prompt, up to 3 concurrent generations
- **Generation Status Tracking** — real-time status (generating/complete/failed) with error display
- **Generation History** — browse, preview, and bulk-delete past generations
- **Saved Prompts & References** — store and reuse favorites with multi-select in reference library
- **Token Usage & Cost Estimates** — see prompt tokens and estimated cost per generation
- **Quota Management** — friendly error messages and UI banner when daily API limits are reached

## Tech Stack

- **Frontend**: Next.js 16, React 19, TypeScript
- **Backend**: [Convex](https://convex.dev) (real-time database + serverless functions)
- **AI**: Google Generative AI SDK (`@google/genai`)
- **Styling**: Tailwind CSS v4, shadcn/ui, tw-animate-css

## Prerequisites

- [Node.js](https://nodejs.org/) v18 or later
- A free [Convex](https://convex.dev) account
- A [Google AI API key](https://aistudio.google.com/apikey) (free tier: 250 requests/day) **or** a Google Cloud project with Vertex AI enabled (pay-as-you-go, no daily cap)

## Getting Started

### 1. Clone the repo

```bash
git clone <your-repo-url>
cd tarkan
```

### 2. Install dependencies

```bash
npm install
```

### 3. Set up Convex

Create a free account at [convex.dev](https://convex.dev) if you don't have one, then initialize a new Convex project:

```bash
npx convex dev
```

On first run, this will:

- Prompt you to log in to Convex
- Ask you to create a new project (or link an existing one)
- Generate a `.env.local` file with your project's `CONVEX_DEPLOYMENT` and `NEXT_PUBLIC_CONVEX_URL`
- Push the database schema and functions to your Convex backend
- Start watching for changes

> Keep this terminal running — it syncs your `convex/` directory to the cloud on every save.

### 4. Set your API credentials

**Option A: Google AI Studio (free, 250 requests/day)**

Get an API key from [Google AI Studio](https://aistudio.google.com/apikey):

```bash
npx convex env set GOOGLE_AI_API_KEY <your-key-here>
```

**Option B: Vertex AI (pay-as-you-go, no daily cap)**

If you hit the free-tier limit, switch to Vertex AI with a GCP service account:

1. Enable the Vertex AI API in your [Google Cloud project](https://console.cloud.google.com)
2. Create a service account with `roles/aiplatform.user` and download the JSON key
3. Set the env vars:

```bash
npx convex env set GOOGLE_CLOUD_PROJECT <your-gcp-project-id>
npx convex env set GOOGLE_CLOUD_CREDENTIALS '<paste-entire-service-account-json>'
npx convex env set GOOGLE_CLOUD_LOCATION global
```

When both are set, Vertex AI takes precedence. Unset `GOOGLE_CLOUD_PROJECT` and `GOOGLE_CLOUD_CREDENTIALS` to revert to AI Studio.

All keys are used server-side by Convex actions — they are **not** stored in `.env.local`.

### 5. Start the dev server

In a **second terminal** (keep `npx convex dev` running in the first):

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) and start generating images.

## Environment Variables

| Variable | Where | Required | Description |
|---|---|---|---|
| `NEXT_PUBLIC_CONVEX_URL` | `.env.local` | Yes | Your Convex deployment URL (auto-generated by `npx convex dev`) |
| `CONVEX_DEPLOYMENT` | `.env.local` | Yes | Your Convex deployment identifier (auto-generated by `npx convex dev`) |
| `GOOGLE_AI_API_KEY` | Convex Dashboard | Yes* | Google AI Studio API key |
| `GOOGLE_CLOUD_PROJECT` | Convex Dashboard | No | GCP project ID (enables Vertex AI mode) |
| `GOOGLE_CLOUD_CREDENTIALS` | Convex Dashboard | No | Service account JSON key (required with `GOOGLE_CLOUD_PROJECT`) |
| `GOOGLE_CLOUD_LOCATION` | Convex Dashboard | No | GCP region (defaults to `us-central1`, use `global` for preview models) |

\* Required unless using Vertex AI (`GOOGLE_CLOUD_PROJECT` + `GOOGLE_CLOUD_CREDENTIALS`). Set via `npx convex env set` or in the [Convex Dashboard](https://dashboard.convex.dev) under Settings > Environment Variables.

## AI Models

| Model | Best For | Notes |
|---|---|---|
| **Imagen 4** | Text-to-image | Highest quality, supports aspect ratio control. Falls back to Gemini 3 Pro when a reference image is provided. |
| **Nano Banana Pro** (Gemini 3 Pro) | Reference-guided generation | Best quality when using a reference image |
| **Nano Banana 2** (Gemini 3.1 Flash) | Fast generation | Fastest model, good for high-volume use |
| **Nano Banana OG** (Gemini 2.5 Flash) | Stable/GA | Cheapest model, fully stable |
