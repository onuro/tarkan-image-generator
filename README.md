# Tarkan вЂ” AI Image Generator

A full-stack AI image generation app built with Next.js, Convex, and Google's Gemini/Imagen models. Generate images from text prompts with style presets, reference images, prompt enhancement, and more.

## Features

- **Text-to-Image Generation** with multiple Google AI models (Imagen 4, Gemini 3 Pro, Gemini 3.1 Flash)
- **Prompt Enhancement** вЂ” auto-expands short prompts using Gemini 2.5 Flash
- **Multi-Reference Images** вЂ” attach up to 14 reference images per generation (upload, paste, or pick from saved library)
- **@Mention Autocomplete** вЂ” type `@` in the prompt to reference attached images by label (`@img1`, `@img2`, etc.) with thumbnail previews
- **11 Style Presets** вЂ” photorealistic, 3D render, anime, oil painting, watercolor, cinematic, and more
- **Aspect Ratio Control** вЂ” auto, 1:1, 16:9, 9:16, 4:3, 3:4
- **Batch Generation** вЂ” generate 1вЂ“4 images per prompt, up to 3 concurrent generations
- **Generation Status Tracking** вЂ” real-time status (generating/complete/failed) with error display
- **Generation History** вЂ” browse, preview, and bulk-delete past generations
- **Saved Prompts & References** вЂ” store and reuse favorites with multi-select in reference library
- **Token Usage & Cost Estimates** вЂ” see prompt tokens and estimated cost per generation
- **Quota Management** вЂ” friendly error messages and UI banner when daily API limits are reached

## Tech Stack

- **Frontend**: Next.js 16, React 19, TypeScript
- **Backend**: [Convex](https://convex.dev) (real-time database + serverless functions)
- **AI**: Google Generative AI SDK (`@google/genai`)
- **Styling**: Tailwind CSS v4, shadcn/ui, tw-animate-css

## Prerequisites

- [Node.js](https://nodejs.org/) v18 or later
- A free [Convex](https://convex.dev) account
- A [Google AI API key](https://aistudio.google.com/apikey)

## Getting Started

### 1. Clone the repo

```bash
git clone <your-repo-url>
cd tarkan
```

### 2. Install dependencies

```bash
npm install
```

### 3. Set up Convex

Create a free account at [convex.dev](https://convex.dev) if you don't have one, then initialize a new Convex project:

```bash
npx convex dev
```

On first run, this will:

- Prompt you to log in to Convex
- Ask you to create a new project (or link an existing one)
- Generate a `.env.local` file with your project's `CONVEX_DEPLOYMENT` and `NEXT_PUBLIC_CONVEX_URL`
- Push the database schema and functions to your Convex backend
- Start watching for changes

> Keep this terminal running вЂ” it syncs your `convex/` directory to the cloud on every save.

### 4. Set your Google AI API key

Get an API key from [Google AI Studio](https://aistudio.google.com/apikey), then set it as a Convex environment variable:

```bash
npx convex env set GOOGLE_AI_API_KEY <your-key-here>
```

This key is used server-side by Convex actions вЂ” it is **not** stored in `.env.local`.

### 5. Start the dev server

In a **second terminal** (keep `npx convex dev` running in the first):

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) and start generating images.

## Environment Variables

| Variable | Where | Required | Description |
|---|---|---|---|
| `NEXT_PUBLIC_CONVEX_URL` | `.env.local` | Yes | Your Convex deployment URL (auto-generated by `npx convex dev`) |
| `CONVEX_DEPLOYMENT` | `.env.local` | Yes | Your Convex deployment identifier (auto-generated by `npx convex dev`) |
| `GOOGLE_AI_API_KEY` | Convex Dashboard | Yes | Google AI API key вЂ” set via `npx convex env set` or in the [Convex Dashboard](https://dashboard.convex.dev) under Settings > Environment Variables |

## AI Models

| Model | Best For | Notes |
|---|---|---|
| **Imagen 4** | Text-to-image | Highest quality, supports aspect ratio control. Falls back to Gemini 3 Pro when a reference image is provided. |
| **Nano Banana Pro** (Gemini 3 Pro) | Reference-guided generation | Best quality when using a reference image |
| **Nano Banana 2** (Gemini 3.1 Flash) | Fast generation | Fastest model, good for high-volume use |
